# 01 — Conversion-Optimised Copywriting Prompt

> **Goal:** Rewrite every word on the site so it speaks directly to the ideal client, builds desire, handles objections, and drives one clear action per page.

---

## Pre-Work — Scan First

Before writing a single word, do the following:

1. Read every page of the current site
2. List every headline, subheadline, CTA, body paragraph, and label
3. Identify the **single ideal client** this site is for (their role, pain, desire)
4. Identify the **one action** each page wants the visitor to take

Output this analysis before writing anything.

---

## Copywriting Frameworks to Apply

### Homepage Hero — StoryBrand Formula
```
Character (client) + Problem + Guide (you) + Plan + Call to Action + Success/Failure stakes
```

**Hero headline formula:**
> "[What you do] for [who] so they can [desired outcome]"

**NOT:**
> "Welcome to our website" / "We are a creative studio" / "Innovative solutions"

**YES:**
> "We Build Websites That Win Clients While You Sleep"
> "Turn Your Portfolio Into a 24/7 Sales Machine"

---

### Body Copy — PAS Formula
Apply to every feature/service description:

1. **P — Problem:** Name the exact pain the client feels
2. **A — Agitate:** Make it vivid. Make them feel it.
3. **S — Solution:** Position your offer as the relief

---

### CTA Copy — AIDA Formula
Every call-to-action must:

1. **A — Attention:** Strong verb that implies transformation
2. **I — Interest:** Hint at what happens next
3. **D — Desire:** Imply the outcome
4. **A — Action:** One clear, frictionless step

**NOT:** "Submit" / "Click Here" / "Learn More"

**YES:** "Get My Free Strategy Session" / "See How It Works" / "Start Building Today"

---

## Page-by-Page Copy Audit

### Homepage
Rewrite in this order:

**Hero Section**
- [ ] Headline: outcome-focused, specific, memorable (8–12 words max)
- [ ] Subheadline: expands on headline, names the client, names the result (1–2 sentences)
- [ ] Primary CTA: action verb + outcome (3–5 words)
- [ ] Secondary CTA: lower commitment option (3–5 words)
- [ ] Social proof line below hero: "Trusted by X clients / X projects delivered"

**Features/Services Section**
- [ ] Section headline: benefit-led, not feature-led
- [ ] Each card headline: starts with a verb or outcome
- [ ] Each card description: 2 sentences max — problem solved + how

**About/Story Section**
- [ ] Opens with the client's problem, not your company history
- [ ] Positions you as the guide, not the hero
- [ ] Ends with a transition to the next section

**CTA Section**
- [ ] High-stakes headline: what they gain / what they lose by waiting
- [ ] One CTA button only
- [ ] Optional: risk-reversal line ("No commitment. Cancel anytime.")

**Footer**
- [ ] Tagline: memorable, 5 words max
- [ ] All link labels are clear and action-oriented

---

### Other Pages

**About Page**
- [ ] Opens with client empathy, not founder bio
- [ ] Story arc: you noticed a problem → you built a solution → here's proof
- [ ] Team bios: skill + personality + what they love doing for clients

**Services/Work Page**
- [ ] Each service headline: outcome, not deliverable ("Launch in 4 weeks" not "Web Development")
- [ ] Process section: numbered steps, plain language, no jargon
- [ ] Each case study: Before → What we did → After (with numbers)

**Contact Page**
- [ ] Headline reduces friction: "Let's Talk" not "Contact Us"
- [ ] Subheadline sets expectations: response time, what happens next
- [ ] Form labels are conversational, not clinical

---

## Copy Rules — Apply Everywhere

| Rule | Bad Example | Good Example |
|------|-------------|--------------|
| Specific beats vague | "Great results" | "47% more leads in 90 days" |
| Client first | "We specialise in..." | "You need a site that..." |
| Short sentences | Long winding clauses | Max 20 words per sentence |
| Active voice | "Solutions are delivered by us" | "We deliver solutions" |
| No jargon | "Leveraging synergistic frameworks" | "We build things that work" |
| One idea per paragraph | Multiple points crammed in | One point, then line break |
| Numbers over adjectives | "Many happy clients" | "127 clients launched" |

## Tone of Voice Guidelines

Define and apply one consistent voice across the entire site:

```
TONE:       [e.g. Confident but warm / Minimal and sharp / Bold and playful]
NEVER SAY:  "Solutions", "Leverage", "Synergy", "Innovative", "World-class"
ALWAYS SAY: [your specific brand vocabulary — define 5–10 owned words/phrases]
READING AGE: Grade 8 or below (use Hemingway App to check)
SENTENCE LENGTH: Max 20 words average
PARAGRAPH LENGTH: Max 3 sentences
```

# 02 — Visual Storytelling & Scroll Journey Prompt

> **Goal:** Transform the homepage into a scroll-driven narrative. Every section is a chapter. The visitor is guided emotionally from problem to solution to desire to action — without realising they're being sold to.

---

## The Scroll Story Arc

Map the homepage to this emotional journey:

```
SCROLL POSITION    EMOTION           SECTION
─────────────────────────────────────────────
0% (Hero)       →  Intrigue          "This is for me"
15%             →  Recognition       "They understand my problem"
30%             →  Credibility       "These people are legit"
50%             →  Desire            "I want what they have"
65%             →  Trust             "Others like me chose them"
80%             →  Urgency           "I should act now"
100% (Footer)   →  Ease              "Starting is simple"
```

Redesign each section to deliberately trigger its target emotion.

---

## Scroll Techniques to Implement

### 1. Scroll-Triggered Reveals

Every section animates in as it enters the viewport. No static page loads.

```tsx
// Framer Motion scroll trigger pattern
import { motion, useInView } from 'framer-motion'
import { useRef } from 'react'

const ScrollReveal = ({ children, delay = 0 }) => {
  const ref = useRef(null)
  const isInView = useInView(ref, { once: true, margin: '-100px' })

  return (
    <motion.div
      ref={ref}
      initial={{ opacity: 0, y: 40 }}
      animate={isInView ? { opacity: 1, y: 0 } : {}}
      transition={{ duration: 0.7, delay, ease: [0.22, 1, 0.36, 1] }}
    >
      {children}
    </motion.div>
  )
}
```

Apply staggered delays to child elements within each section (0, 0.1, 0.2, 0.3s).

---

### 2. Parallax Backgrounds

Hero and CTA sections should have parallax depth layers:

```tsx
import { useScroll, useTransform, motion } from 'framer-motion'

const ParallaxHero = () => {
  const { scrollY } = useScroll()
  const y = useTransform(scrollY, [0, 500], [0, 150])
  const opacity = useTransform(scrollY, [0, 400], [1, 0])

  return (
    <section style={{ overflow: 'hidden', position: 'relative' }}>
      <motion.div style={{ y, opacity }} className="hero-bg-image" />
      <div className="hero-content">/* content stays fixed */</div>
    </section>
  )
}
```

---

### 3. Sticky Scroll Sections

Create at least one "sticky" storytelling section where text changes as user scrolls, while an image stays pinned:

```
┌─────────────────────────────────────┐
│  [Sticky Image]  │  [Scrolling text] │
│                  │  Chapter 1...     │
│  (stays pinned   │  ─────────────── │
│   while text     │  Chapter 2...     │
│   scrolls by)    │  ─────────────── │
│                  │  Chapter 3...     │
└─────────────────────────────────────┘
```

```tsx
// Sticky scroll implementation
<div className="sticky-scroll-container" style={{ display: 'grid', gridTemplateColumns: '1fr 1fr' }}>
  <div style={{ position: 'sticky', top: '10vh', height: '80vh' }}>
    {/* Image that morphs based on scroll progress */}
  </div>
  <div>
    {chapters.map((chapter) => (
      <div style={{ minHeight: '100vh', display: 'flex', alignItems: 'center' }}>
        {chapter.content}
      </div>
    ))}
  </div>
</div>
```

---

### 4. Counter Animations

Stats and numbers animate up when scrolled into view:

```tsx
const AnimatedCounter = ({ target, suffix = '' }) => {
  const ref = useRef(null)
  const isInView = useInView(ref, { once: true })
  const count = useMotionValue(0)
  const rounded = useTransform(count, Math.round)

  useEffect(() => {
    if (isInView) {
      animate(count, target, { duration: 2, ease: 'easeOut' })
    }
  }, [isInView])

  return <motion.span ref={ref}>{rounded}{suffix}</motion.span>
}

// Usage: <AnimatedCounter target={127} suffix="+" />
```

---

### 5. Image Reveal on Scroll

Gallery and portfolio images sweep in with a clip-path wipe:

```tsx
const ImageReveal = ({ src, alt }) => {
  const ref = useRef(null)
  const isInView = useInView(ref, { once: true })

  return (
    <motion.div
      ref={ref}
      initial={{ clipPath: 'inset(0 100% 0 0)' }}
      animate={isInView ? { clipPath: 'inset(0 0% 0 0)' } : {}}
      transition={{ duration: 0.9, ease: [0.76, 0, 0.24, 1] }}
    >
      <Image src={src} alt={alt} fill />
    </motion.div>
  )
}
```

---

### 6. Horizontal Scroll Section (Optional Premium Touch)

One section scrolls horizontally instead of vertically — great for showcasing project work:

```tsx
const HorizontalScroll = ({ items }) => {
  const ref = useRef(null)
  const { scrollYProgress } = useScroll({ target: ref })
  const x = useTransform(scrollYProgress, [0, 1], ['0%', '-75%'])

  return (
    <section ref={ref} style={{ height: '400vh' }}>
      <div style={{ position: 'sticky', top: 0, overflow: 'hidden' }}>
        <motion.div style={{ x, display: 'flex' }}>
          {items.map(item => <div style={{ minWidth: '40vw' }}>{item}</div>)}
        </motion.div>
      </div>
    </section>
  )
}
```

---

## Section-by-Section Scroll Design

### Hero (0%)
- Full viewport height minimum
- Text fades in on load with stagger (headline → subheadline → CTA)
- Scroll indicator (animated arrow or "scroll" text) fades in after 2s
- Background image/video has parallax drift

### Problem Section (15%)
- Headline scrolls in from left
- 3 pain point cards stagger in from bottom
- Each card has a subtle pulse or shimmer on appear

### Credibility (30%)
- Logo strip scrolls horizontally (infinite marquee)
- Stats counters animate up
- Awards/features badges pop in sequentially

### Work/Gallery (50%)
- Images use clip-path wipe reveal
- Hover reveals project details with smooth overlay
- If horizontal scroll: use the pinned horizontal technique

### Testimonials (65%)
- Cards scroll in alternating left/right
- Active card is slightly larger (scale 1.05)
- Author photo fades in after card appears

### CTA (80%)
- Background shifts colour or gradient on scroll into view
- Headline splits and animates word by word
- Button has magnetic pull effect on hover

---

## Performance Rules for Animations

- [ ] All animations use `transform` and `opacity` only (no `top`, `left`, `width`)
- [ ] Use `will-change: transform` sparingly and only on actively animating elements
- [ ] Implement `prefers-reduced-motion` media query to disable all animations for users who need it
- [ ] Test scroll performance on mobile (must be 60fps)
- [ ] No animation longer than 1 second (except page transitions)

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

# 03 — Typography & Whitespace Mastery Prompt

> **Goal:** Make the typography itself a design element. Perfect spacing, perfect rhythm, perfect hierarchy — so beautiful it could be printed and framed.

---

## Step 1 — Choose the Font Pairing

Pick ONE display font + ONE body font. They must feel intentional together.

**Requirements:**
- Display font: distinctive, has personality, memorable, loads from Google Fonts or is a web-safe variable font
- Body font: highly legible at small sizes, pairs well with display
- NOT Inter. NOT Roboto. NOT Arial. NOT system-ui.

**Strong pairings to consider:**

| Display | Body | Feel |
|---------|------|------|
| Playfair Display | DM Sans | Editorial luxury |
| Space Grotesk | Lora | Modern meets classic |
| Syne | Nunito | Creative & approachable |
| Bebas Neue | Source Sans 3 | Bold & clean |
| Cormorant Garamond | Jost | High-fashion editorial |
| Cabinet Grotesk | Literata | Premium contemporary |
| Archivo Black | Archivo | Strong & unified |

**Implement with `next/font`:**

```tsx
// app/layout.tsx
import { Playfair_Display, DM_Sans } from 'next/font/google'

const display = Playfair_Display({
  subsets: ['latin'],
  variable: '--font-display',
  weight: ['400', '700', '900'],
})

const body = DM_Sans({
  subsets: ['latin'],
  variable: '--font-body',
  weight: ['300', '400', '500', '600'],
})

export default function RootLayout({ children }) {
  return (
    <html className={`${display.variable} ${body.variable}`}>
      <body>{children}</body>
    </html>
  )
}
```

```css
  --text-lg:   1.563rem;   /* 25px */
  --text-xl:   1.953rem;   /* 31px */
  --text-2xl:  2.441rem;   /* 39px */
  --text-3xl:  3.052rem;   /* 49px */
  --text-4xl:  3.815rem;   /* 61px */
```

## Step 4 — Whitespace System (The 8-Point Grid)

Every spacing value must be a multiple of 8px. No exceptions.

```css
:root {
  --space-1:  8px;    /* Tight: icon gap, label padding */
  --space-2:  16px;   /* Compact: input padding, list gap */
  --space-3:  24px;   /* Default: component padding */
  --space-4:  32px;   /* Medium: section sub-spacing */
  --space-6:  48px;   /* Large: between components */
  --space-8:  64px;   /* XL: section padding */
  --space-12: 96px;   /* XXL: major section breaks */
  --space-16: 128px;  /* Hero vertical padding */
  --space-24: 192px;  /* Maximum section padding */
}
```

**Rules:**
- Section vertical padding: minimum `--space-12` (96px) on desktop
- Between sections: `--space-8` to `--space-16`
- Between a heading and its body text: `--space-2` (16px)
- Between a heading and the section above it: `--space-4` (32px)
- Card internal padding: `--space-3` to `--space-4`
- Never use arbitrary values like `mt-[37px]` — only scale values

# 04 — Micro-Interactions & Animation Prompt

> **Goal:** Every interactive element feels alive and tactile. Hover states, button presses, link underlines, image hovers, menu opens — nothing is static. Reference Awwwards winners for the level of craft required.

---

## Philosophy

> "Animation is not decoration. It's communication."

Every animation must have a **purpose**:
- **Feedback**: confirms the user did something
- **Guidance**: shows where to go next
- **Delight**: rewards attention and exploration
- **Narrative**: tells the story of the interface

If an animation has none of these purposes — remove it.

---

## Animation Principles

| Principle | Rule |
|-----------|------|
| Duration | 150–400ms for micro, 600–900ms for reveals |
| Easing | Always custom cubic-bezier, never `ease` or `linear` |
| Distance | Elements move max 40px — subtle, not dramatic |
| Scale | Hover scale max 1.05 — more feels broken |
| Stagger | Child elements: 60–80ms apart |
| GPU | Only animate `transform` and `opacity` |

**Custom easings to use:**
```css
:root {
  --ease-out-expo:    cubic-bezier(0.16, 1, 0.3, 1);
  --ease-in-out-expo: cubic-bezier(0.87, 0, 0.13, 1);
  --ease-spring:      cubic-bezier(0.34, 1.56, 0.64, 1);  /* slight overshoot */
  --ease-smooth:      cubic-bezier(0.22, 1, 0.36, 1);
}
```

---

## 1. Button Micro-Interactions

### Primary Button — Full suite of states

```tsx
// components/ui/Button.tsx
import { motion } from 'framer-motion'

const Button = ({ children, onClick, variant = 'primary' }) => {
  return (
    <motion.button
      className={`btn btn-${variant}`}
      onClick={onClick}
      whileHover={{ scale: 1.03 }}
      whileTap={{ scale: 0.97 }}
      transition={{ type: 'spring', stiffness: 400, damping: 25 }}
    >
      <span className="btn-text">{children}</span>
      <span className="btn-arrow" aria-hidden>→</span>
    </motion.button>
  )
}
```


```tsx
const MagneticButton = ({ children }) => {
  const ref = useRef(null)
  const x = useMotionValue(0)
  const y = useMotionValue(0)

  const handleMouse = (e) => {
    const rect = ref.current.getBoundingClientRect()
    const centerX = rect.left + rect.width / 2
    const centerY = rect.top + rect.height / 2
    x.set((e.clientX - centerX) * 0.3)
    y.set((e.clientY - centerY) * 0.3)
  }

  const handleLeave = () => {
    x.set(0)
    y.set(0)
  }

  return (
    <motion.button
      ref={ref}
      style={{ x, y }}
      onMouseMove={handleMouse}
      onMouseLeave={handleLeave}
      transition={{ type: 'spring', stiffness: 150, damping: 15 }}
    >
      {children}
    </motion.button>
  )
}
```

## Theme Toggle Component

```tsx
// components/ui/ThemeToggle.tsx
'use client'
import { useEffect, useState } from 'react'
import { motion, AnimatePresence } from 'framer-motion'

export const ThemeToggle = () => {
  const [theme, setTheme] = useState<'light' | 'dark'>('light')
  const [mounted, setMounted] = useState(false)

  useEffect(() => {
    setMounted(true)
    const stored = localStorage.getItem('theme')
    const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches
    const initial = stored ?? (prefersDark ? 'dark' : 'light')
    setTheme(initial as 'light' | 'dark')
    document.documentElement.setAttribute('data-theme', initial)
  }, [])

  const toggle = () => {
    const next = theme === 'light' ? 'dark' : 'light'
    setTheme(next)
    document.documentElement.setAttribute('data-theme', next)
    localStorage.setItem('theme', next)
  }

  if (!mounted) return <div style={{ width: 44, height: 28 }} /> // prevent hydration flash

  return (
    <motion.button
      onClick={toggle}
      aria-label={`Switch to ${theme === 'light' ? 'dark' : 'light'} mode`}
      className="theme-toggle"
      whileTap={{ scale: 0.9 }}
    >
      <AnimatePresence mode="wait">
        {theme === 'light' ? (
          <motion.span
            key="moon"
            initial={{ rotate: -90, opacity: 0 }}
            animate={{ rotate: 0, opacity: 1 }}
            exit={{ rotate: 90, opacity: 0 }}
            transition={{ duration: 0.2 }}
          >
            🌙
          </motion.span>
        ) : (
          <motion.span
            key="sun"
            initial={{ rotate: 90, opacity: 0 }}
            animate={{ rotate: 0, opacity: 1 }}
            exit={{ rotate: -90, opacity: 0 }}
            transition={{ duration: 0.2 }}
          >
            ☀️
          </motion.span>
        )}
      </AnimatePresence>
    </motion.button>
  )
}
```

## Dark Mode Design Principles

### What makes dark mode feel premium (not lazy):

| Bad Dark Mode | Premium Dark Mode |
|---------------|------------------|
| `background: #000000` | Rich dark: `#0A0A0A` or `#0D0D0D` |
| `color: #FFFFFF` | Soft white: `#F5F5F5` or `#EDEDED` |
| Same shadows as light | Shadows removed, glows added |
| Same accent color | Accent lightened 15–20% |
| Flat surfaces | Subtle elevation hierarchy via bg shades |
| Gray borders | Near-invisible borders: `#2A2A2A` |

## Detail 1 — Custom Cursor

```tsx
// components/ui/CustomCursor.tsx
'use client'
import { useEffect, useRef, useState } from 'react'
import { motion, useMotionValue, useSpring } from 'framer-motion'

type CursorState = 'default' | 'hover' | 'click' | 'text' | 'image'

export const CustomCursor = () => {
  const cursorX = useMotionValue(-100)
  const cursorY = useMotionValue(-100)
  const [state, setState] = useState<CursorState>('default')
  const [label, setLabel] = useState('')

  // Smooth spring following
  const springX = useSpring(cursorX, { stiffness: 500, damping: 40 })
  const springY = useSpring(cursorY, { stiffness: 500, damping: 40 })

  useEffect(() => {
    const move = (e: MouseEvent) => {
      cursorX.set(e.clientX)
      cursorY.set(e.clientY)
    }

    const handleHover = (e: MouseEvent) => {
      const target = e.target as HTMLElement
      if (target.matches('a, button, [role="button"]')) {
        setState('hover')
        setLabel(target.dataset.cursorLabel || '')
      } else if (target.matches('p, span, h1, h2, h3, h4')) {
        setState('text')
        setLabel('')
      } else if (target.matches('img, [data-cursor="image"]')) {
        setState('image')
        setLabel('View')
      } else {
        setState('default')
        setLabel('')
      }
    }

    window.addEventListener('mousemove', move)
    window.addEventListener('mouseover', handleHover)
    return () => {
      window.removeEventListener('mousemove', move)
      window.removeEventListener('mouseover', handleHover)
    }
  }, [])

  // Hide on touch devices
  if (typeof window !== 'undefined' && window.matchMedia('(hover: none)').matches) {
    return null
  }

  const variants = {
    default: { width: 12,  height: 12,  backgroundColor: 'var(--color-accent)', mixBlendMode: 'normal' },
    hover:   { width: 48,  height: 48,  backgroundColor: 'var(--color-accent)', mixBlendMode: 'difference' },
    text:    { width: 4,   height: 28,  borderRadius: 2, backgroundColor: 'var(--color-accent)' },
    image:   { width: 80,  height: 80,  backgroundColor: 'var(--color-accent)', mixBlendMode: 'difference' },
  }

  return (
    <>
      {/* Hide default cursor everywhere */}
      <style>{`* { cursor: none !important; }`}</style>

      <motion.div
        style={{
          position: 'fixed',
          left: springX,
          top: springY,
          x: '-50%',
          y: '-50%',
          borderRadius: '50%',
          pointerEvents: 'none',
          zIndex: 9999,
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
        }}
        animate={variants[state]}
        transition={{ type: 'spring', stiffness: 400, damping: 30 }}
      >
        {label && (
          <span style={{ color: 'white', fontSize: 12, fontWeight: 600 }}>
            {label}
          </span>
        )}
      </motion.div>
    </>
  )
}
```

## Detail 6 — Text Scramble Effect

For hero headlines — letters scramble before revealing the final word:

```tsx
// hooks/useTextScramble.ts
export const useTextScramble = (text: string, trigger: boolean) => {
  const [display, setDisplay] = useState('')
  const chars = 'ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789'

  useEffect(() => {
    if (!trigger) return

    let iteration = 0
    const interval = setInterval(() => {
      setDisplay(
        text.split('').map((char, i) => {
          if (i < iteration) return char
          if (char === ' ') return ' '
          return chars[Math.floor(Math.random() * chars.length)]
        }).join('')
      )
      iteration += 0.5
      if (iteration >= text.length) clearInterval(interval)
    }, 30)

    return () => clearInterval(interval)
  }, [text, trigger])

  return display || text
}

// Usage
const HeroHeadline = () => {
  const ref = useRef(null)
  const isInView = useInView(ref, { once: true })
  const text = useTextScramble('We Build Digital Experiences', isInView)
  return <h1 ref={ref}>{text}</h1>
}
```

### Keyboard Trap Fix for Modal/Menu

```tsx
// hooks/useFocusTrap.ts
export const useFocusTrap = (ref: RefObject<HTMLElement>, active: boolean) => {
  useEffect(() => {
    if (!active || !ref.current) return

    const focusable = ref.current.querySelectorAll(
      'a, button, input, select, textarea, [tabindex]:not([tabindex="-1"])'
    )
    const first = focusable[0] as HTMLElement
    const last = focusable[focusable.length - 1] as HTMLElement

    const handleTab = (e: KeyboardEvent) => {
      if (e.key !== 'Tab') return
      if (e.shiftKey) {
        if (document.activeElement === first) {
          e.preventDefault()
          last.focus()
        }
      } else {
        if (document.activeElement === last) {
          e.preventDefault()
          first.focus()
        }
      }
    }

    document.addEventListener('keydown', handleTab)
    first?.focus()

    return () => document.removeEventListener('keydown', handleTab)
  }, [active])
}
```

## Step 3 — Fix CLS (Layout Shift)

Layout shift happens when content moves after initial render:

```tsx
// ❌ Image without dimensions — causes layout shift
<img src="photo.jpg" />

// ✅ Always specify dimensions or use fill with a sized container
<div style={{ position: 'relative', aspectRatio: '16/9' }}>
  <Image src="photo.jpg" alt="" fill />
</div>

// ❌ Font loading without fallback — causes text shift
// (Using next/font fixes this automatically)

// ❌ Dynamically injected content that pushes layout
// Fix: Reserve space with min-height before content loads
<div style={{ minHeight: '200px' }}>
  {content ?? <Skeleton />}
</div>
```

```css
/* Prevent image layout shift with aspect-ratio */
.image-container {
  aspect-ratio: 16 / 9;
  position: relative;
  overflow: hidden;
}

/* Reserve space for fonts */
.hero-headline {
  /* Use size-adjust in @font-face to match fallback metrics */
  font-family: var(--font-display), Georgia;
}
```
